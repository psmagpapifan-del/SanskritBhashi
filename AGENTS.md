# App Release Notes Management
Whenever you implement bug fixes, performance improvements, UI adjustments, or new features in the codebase:
1. You MUST document them by appending a concise summary of the change to the `RELEASE_NOTES.md` file in the project root.
2. The summary must include the date, change type (Bug Fix, Feature, Performance, UI), file(s) affected, and a brief description.
3. This ensures all modifications are properly tracked and ready to be compiled into the next Google Play Store app release notes.
